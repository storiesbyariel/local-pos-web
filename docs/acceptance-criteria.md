# Acceptance Criteria (Milestone 0)

1. User can create, edit, and delete catalog items with name + unit price.
2. User can add catalog items to cart and modify qty/unit price per line.
3. User can set discount type/value and one tax rate.
4. Totals recalculate correctly: subtotal, discount, tax, grand total.
5. User can checkout as Cash or Card.
6. Latest receipt shows transaction details and totals.
7. Transaction history lists completed checkouts in reverse chronological order.
8. Data persists after page refresh and browser reopen.
9. App runs as static frontend with no backend dependencies.
10. Unit tests validate pricing math; at least one flow/integration-style test validates end-to-end state flow.
